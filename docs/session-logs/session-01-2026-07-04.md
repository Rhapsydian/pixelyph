## Goal

Define what Pixelyph is, nail down the architecture, choose the technology stack, and produce a written plan that each subsequent implementation session could follow independently.

## Key Decisions

**What Pixelyph is**
- Two modes: **Draw mode** (traditional pixel art, multi-color, layered) and **Glyph mode** (per-codepoint boolean grids for real font and icon-font output)
- Output is always SVG/vector via `pixelloom`, not raster — raster is a secondary export via Canvas2D

**Architecture**
- The editing surface is **real SVG DOM**, not Canvas2D — the same `composeLayersSvg`/`gridToPath` pipeline used for export. This guarantees WYSIWYG: what you see while drawing is exactly what gets exported, gradients and all.
- Each `Layer` is single-color, mapping directly to one `gridToPath` call. `Canvas` holds `Layer[]`; each layer has a `LayerStyle` and `frames: Frame[]` (length 1 in v1, animation deferred).
- **Simple tier**: one conceptual canvas, auto-managed layers hidden from the user. `autoLayerSync` reconciles multi-color per-cell painting into the correct per-color layers transparently.
- **Advanced tier**: independently selectable layers with per-layer fill (solid/gradient), stroke, and effects.

**Technology stack**
- Plain JavaScript + JSDoc (`tsc --noEmit` for editor checking only, no TypeScript build step) — consistent with pixelloom's own conventions
- React (over user's preferred Svelte) for ecosystem breadth
- Vite as dev server and bundler
- `zustand` for state management
- `node --test` for testing — zero-config, same as pixelloom
- `opentype.js` for font compilation; `ttf2woff`/`wawoff2` for WOFF/WOFF2
- Electron (not Tauri) for the eventual desktop shell (Phase 6)
- Hard rule: `src/model/`, `src/export/`, `src/io/projectFile.js` must never import React, zustand, or any browser/Node API — pure functions only

**Repo and dependency management**
- New standalone repo at `C:\Users\husbando\Claude\pixelyph`, not a monorepo
- `pixelloom` referenced via semver in `package.json`; local iteration uses `npm link ../pixelloom` (never a committed `file:` path)

**Animation**
- Frames baked into the data model from day one (`layers[].frames: Frame[]`, length 1 default) to avoid a costly migration later; animation UI deferred to Phase 7
