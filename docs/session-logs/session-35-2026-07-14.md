## Key decisions and corrections (with rationale)

1. **Manual rewrite depth was clarified before starting**, since the
   existing pages were already reasonably-written reference prose, not
   broken — the open question was how far to restructure. The user picked
   a full tutorial rewrite (goal-oriented walkthroughs, jargon defined on
   first use) over a lighter cleanup pass; `keyboard-shortcuts.md` was
   scoped out from the start since a shortcut table is already the right
   format for reference lookup.
2. **The WOFF2 export caveat was dropped entirely, not reworded.** The
   plan's first draft replaced `export.md`'s pointer to `BACKLOG.md` (a
   dev-facing file, wrong to link from a user manual) with a
   "temporarily unavailable" note. The user corrected this at
   `ExitPlanMode` review: WOFF2 is disabled at the source and its checkbox
   isn't even shown in the UI, so there's nothing for a new user to
   encounter — no need to mention a feature that isn't reachable until it
   actually works.
3. **Sample Projects menu placement was adjusted from the proposed
   default.** The plan proposed inserting the submenu between "Open
   Project…" and "Save Project"; the user asked for it after "Save
   Project" instead.
4. **Implementation reused two existing patterns rather than inventing
   new plumbing**: the User Manual's relative-`fetch()`-from-`public/`
   convention (works identically under the dev server, Electron's packaged
   `file://` load, and the GitHub Pages subpath build) for bundling the
   two sample `.pixelyph` files, and a shared `installOpenedDoc` helper
   extracted out of `openAnyProject` so the new `openSampleProject` action
   doesn't duplicate its kind-dispatching install logic. No Electron/IPC
   changes were needed.
5. **`MenuBar.jsx` gained its first nested-flyout component** (`SubMenu`)
   — no "Recent Projects"-style precedent existed anywhere in the app, so
   the hover-to-open-a-second-flyout interaction was built from scratch,
   matching the existing top-level `Menu` dropdown's visual style.
6. **Refinement 1: the Glyph-mode context bar (symmetry/grid/display-color/
   tile) was disappearing whenever no glyph was selected**, because the
   whole row was gated on `glyphCanvas` truthiness. Root-caused as
   over-broad gating: grid and tile are canvas-level settings with no
   per-glyph dependency at all, so they never needed to hide. Fixed by
   always rendering the row and instead `disabled`-ing just the two
   symmetry buttons when there's no active glyph (`setSymmetryMode`
   already no-ops in that case, so this makes the existing behavior
   visible instead of introducing new logic).
7. **Refinement 2: Specimen Preview's insert-glyph swatch buttons were
   spilling their glyph icon and text label outside their own box.**
   Root cause: the global `button { height: var(--control-height) }` rule
   (a fixed 32px, meant for single-line controls) was clamping a button
   whose stacked glyph-SVG-over-label content needs roughly 45px — the
   overflow was rendering, just outside the button's rectangle. Fixed
   with a local `height: 'auto'` override on that button's inline style.
