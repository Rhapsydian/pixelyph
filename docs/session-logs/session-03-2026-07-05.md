# Session 03 — 2026-07-05
**Topic:** Phase 2 — Draw mode, advanced tier  
**Commit:** `af33344` (Phase 2: Draw mode advanced tier)

## Goal

Implement Phase 2 of the plan: the advanced tier of Draw mode — a real layers panel, per-layer fill (solid/gradient), stroke, effects (drop-shadow/blur/glow), eyedropper layer-activation, selection scoping, duplicate/merge-down, CSS-selectable layer IDs in SVG export, and tier conversion.

## Key Decisions

**Shared `composeLayersBody` for WYSIWYG editing**
Rather than maintain two separate markup-generating implementations, `composeLayersSvg.js` was refactored to expose a shared `composeLayersBody` function. The editor injects this via `dangerouslySetInnerHTML`, making the editing surface and the exported SVG structurally byte-identical. This eliminates the maintenance risk of two implementations drifting apart.

**Advanced → simple tier conversion is lossy, with confirmation**
Converting down collapses all layers to one by compositing topmost visible color per cell. Gradients and free-floating positions don't survive. A confirmation dialog is required; the decision was not to try to preserve any of the advanced data.

**Eyedropper activates a layer in advanced tier**
Rather than sampling a color, clicking with the eyedropper tool in advanced mode calls `topVisibleLayerAt` and activates that layer. Rationale: in advanced tier, color is a layer property, not a palette choice — selecting the layer is more useful than sampling the color.

**zustand subscription pattern: subscribe to whole `canvas` object**
`LayersPanel` and `LayerStylePanel` initially subscribed to narrow nested fields (`s.canvas.layers`, `s.canvas.activeLayerId`). Since `canvas` is mutated in place during live operations, narrow selectors produced stale renders. Fixed to subscribe to `s.canvas` — same convention already used by `SvgPixelEditor`.

**Selection scoping toggle**
Added "Select from: Active layer / All visible layers." Cutting in all-visible mode clears each cell from whichever layer actually owns it, preventing orphaned content on non-active layers. Paste always flattens onto the active layer regardless of scope.

**CSS-selectable layer IDs in SVG export**
Each `<g>` in exported SVG gets `id="layer-{slug}"`. Duplicate names get `-2`, `-3` suffixes; unslugifiable names fall back to `layer-unnamed`. Decision: always on — no per-layer toggle — accepted trade-off.

**`duplicateLayer` / `mergeLayerDown`**
Merge-down keeps the bottom layer's id, name, and style — matching Photoshop/Aseprite convention. The merged result is always under the active layer's data.

**Bug fixes discovered during browser verification:**
- **Hover-paints bug:** `handlePointerMove` called `onPointerMove` on every mouse move with no button-down guard — fixed via `isPointerDownRef`
- **Ellipse outline gaps:** `rasterizeEllipse` outline mode collapsed each row to min/max X only, discarding intermediate algorithm points near flat top/bottom of wide ellipses — fixed to emit every traced point
- **Pencil/eraser stale memo:** live drag preview was inside a `useMemo` that never re-ran mid-drag — moved drag rendering out of the memo
- **`liftSelection` missing `tick()` call:** source pixels appeared not cleared mid-drag — fixed

**Follow-on features requested post-verification:**

- Simple-mode palette hidden in advanced tier (replaced with a hint)

- `duplicateLayer` and `mergeLayerDown`
- Ctrl+A selects entire canvas
