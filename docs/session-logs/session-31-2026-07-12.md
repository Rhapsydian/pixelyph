### Checkpoint 3 — Animated PNG (APNG) export (`69ed90a`)

Added `upng-js` for PNG compression and a new `animatedPng.js` mirroring
`animatedRaster.js`'s (GIF) structure. Discovered a real bug in upng-js
itself during implementation: its top-level `UPNG.encode()` pre-sizes its
output buffer as `bufs[0].byteLength * bufs.length + 100` and silently
truncates frames — no error thrown — whenever that estimate undershoots
real per-frame chunk overhead, which reliably happens for small pixel-art
canvases with more than a few frames (verified directly: a 4×4 canvas lost
frames past the third). Worked around it by reusing upng-js's correct
frame compressor (`UPNG.encode.compressPNG`) and exported CRC32/binary-write
helpers, but assembling the actual PNG chunk stream in a custom, unbounded
writer. Extracted `frameSvg`/`loadImageFromBlob` out of `animatedRaster.js`
into a new shared `rasterFrameHelpers.js` so the new file didn't duplicate
them. Five new structural unit tests, plus live-browser verification: fed
the encoder's output into a real `<img>` tag and watched Chrome animate it
(green → red on its own), not just decode it as a static image.

### Checkpoint 4 — In-app User Manual (`06ed856`, `5eadd6b`)

New Help → User Manual modal: a topic sidebar plus a `react-markdown`
(+`remark-gfm` for tables) content pane, fetching plain `.md` files from
`public/manual/` at runtime via relative paths (no leading `/`) so they
resolve correctly under all three ways the app is served — the web
build's GitHub Pages subpath, the local dev server's root, and Electron's
packaged `file://` load — without any bundler-specific path rewriting.
Wrote six topics (Getting Started, Draw Mode, Glyph Mode, Animation,
Export, Keyboard Shortcuts) from the app's actual features/shortcuts, not
invented content. Internal links between topics intercept the click and
switch topic in-app instead of navigating away; last-viewed topic and
scroll position persist to `localStorage`, verified to survive both
modal close/reopen and a full page reload. Root README links directly to
each `public/manual/*.md` file so the manual is readable straight from
GitHub. Real screenshots were explicitly descoped this session — no way
to save actual browser-pane screenshots to disk from this environment
(confirmed: the preview browser isn't a real on-screen window, so no
OS-level capture tool can reach it either) — the image-rendering path
works, but capturing real app screenshots needs a different session
setup. Follow-up fix after user feedback: markdown links rendered in
unstyled browser-default blue/underline, clashing with the app's dark
theme — restyled via a scoped `.manual-content a` rule using the existing
`--chrome-accent`/`--chrome-accent-hover` tokens.

### Checkpoint 5 — Windows CI build + in-app download link (`81f1511`)
