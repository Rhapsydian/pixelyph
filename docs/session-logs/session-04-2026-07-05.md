**Topic:** Phase 3 — Glyph mode (SVG-only)

## Goal

Implement Phase 3 of the plan: a full Glyph mode supporting both character sets (real Unicode codepoints) and icon sets (PUA codepoints, named glyphs), reusing the existing Draw mode pixel editor as much as possible, with per-glyph SVG export and `.pixelyph` project save/load.

## Key Decisions

**Two distinct GlyphSet kinds**
- `'characters'`: maps codepoints to real Unicode characters. The user assigns codepoints via the CharacterMapPanel (typing a character, U+xxxx, etc.).
- `'icons'`: uses Private Use Area codepoints as opaque internal slots. The codepoint is assigned automatically via `nextIconCodepoint()`; the user only sees the glyph name, never the codepoint. Rationale: icon glyphs have no meaningful Unicode identity — treating them as PUA avoids collision with real characters while keeping a uniform codepoint-keyed map for all glyph kinds.

**Monochrome boolean grid (no layers, no color)**
Glyph data is a `Uint8Array` boolean grid — black or transparent, nothing else. The full `Canvas`/`Layer`/`LayerStyle` machinery is unnecessary for font glyphs and would complicate the font-export path. The trade-off (no glyph-level color or effects) is acceptable because font glyphs are always monochrome.

**Reuse over reimplementation: `glyphToCanvas` adapter**
Rather than building a parallel pixel editor, each glyph is wrapped as a single-color, single-layer `'simple'`-tier pseudo-Canvas via `glyphToCanvas` (palette `['#000000']`). `SvgPixelEditor.jsx` — all tools, undo/redo, zoom/pan, grid overlay — needed only a mode-aware document selector, not a rewrite. `GlyphGridEditor.jsx` is literally a one-line wrapper. `canvasToGlyphPixels` reads pixel data back after each stroke commits. This was the central architectural bet of the session.

**Parallel glyph state track in the store**
`glyphSet`, `glyphHistory`, and `activeCodepoint` run alongside the existing Draw-mode state. Switching modes never loses work in either direction. Mode-sensitive actions (`paintCellLive`, `commitStroke`, `undo`, `redo`, `colorAt`) dispatch to the correct document based on `state.mode`.

**Keyboard shortcuts isolated in glyph mode**
Marquee-select and copy-paste keyboard shortcuts are guarded at both the Toolbar (hidden) and the keyboard handler level — not just silently doing nothing. This prevents accidental corruption of the Draw-mode document when in glyph mode.

**Preset design**
Charset presets are pure codepoint lists with no embedded glyph data. Three initial presets: Basic Latin (U+0020–U+007E), Latin-1 Supplement (U+00A0–U+00FF), Digits (U+0030–U+0039). Extended during the session to include a curated Symbols preset (35 codepoints: typographic marks, currency, arrows, card suits, music notes) via multi-select checkbox UI.

**Phase 4 inserted into the roadmap**
During this session, the Phase 4 slot was given to "Project management" (startup screen, New Project wizard, removal of the live mode switcher). Font compilation was renumbered to Phase 5.
