`canvas.tier` used to bundle two independent things into one flag: how many
layers can exist, and how much manual control you get over a shape's style.
The plan splits them — every tier gets real multi-layer support; only
manual shape/style authoring stays tier-gated — and renames the display
labels to **Pixel** (paint colors, shapes auto-managed) and **Shape**
(author shapes manually) now that those are the real axis. The rename is
display-only: `canvas.tier` keeps its stored `'simple'`/`'advanced'` values
everywhere, no save-format bump, no migration.

## Work Completed

### Checkpoint 1 — Retarget `autoLayerSync` to the active layer (`e45c344`)

`autoLayerSync.js`'s `getSimpleLayer` hardcoded `canvas.layers[0]`, so
Simple tier could only ever have one usable layer regardless of how many
existed. Now resolves `canvas.activeLayerId` (falling back to the topmost
layer, then lazy-creating one for a blank canvas). `paintSimpleCell` also
gained lock/per-frame-visibility guards mirroring `paintCell`'s advanced
branch, needed once Checkpoint 3 exposed those toggles on Pixel-tier
layers. Added three new tests: active-layer targeting (painting on a
second, newly-active layer leaves the first layer's own shape untouched),
locked-layer no-op, and hidden-frame no-op.

### Checkpoint 2 — Per-layer `convertTier` collapse + merge-dedup (`683cfc9`)

`convertTier(canvas, 'simple')` used to flatten the *whole canvas* through
cross-layer `colorAt` into a single Layer, discarding layer count, order,
and names. New `collapseLayerToAutoGrids` scans each layer's own frame
independently (`topGridAt`-wins across that layer's own shapes, mirroring
`selection.js`'s `extractRectFromActiveLayer`), so Shape→Pixel now
preserves layer count/order/names/lock/opacity/visibility — only each
layer's own shapes get rebuilt into Pixel tier's one-Grid-per-color form.

Looking ahead to Checkpoint 3 exposing merge on Pixel-tier layers: two
Pixel-tier layers each auto-managing their own same-color Grid would leave
two same-color Grids after a plain concatenation, breaking the
one-Grid-per-color invariant. Added `dedupeSolidColorGrids` to
`mergeLayerDown`, gated to `canvas.tier === 'simple'` only. Factored the
bounding-box + pixel-OR body out of `Grid.js`'s existing `mergeGridDown`
into a shared `unionGridInto(target, source)` primitive that both
`mergeGridDown` and the new dedup logic call.

Rewrote the existing `Canvas.test.js` `convertTier` test (previously
asserted `layers.length === 1` post-collapse, now stale) into three tests:
per-layer collapse preserving layer identity, lock/opacity/visibility
survival, and within-layer same-color merge. Added a new test covering the
merge-dedup fix in both tiers (Pixel-tier same-color shapes fold into one;
Shape-tier same-color shapes stay separate).

### Checkpoint 3 — Pixel-tier Layers panel (`dff8bf3`)

`SidePanel.jsx`'s `drawTabs` only pushed the Layers tab in advanced tier;
now both tiers get it (Style stays Shape-tier-only — manual authoring has
no Pixel-tier equivalent). `LayersPanel.jsx` relaxed its
`tier !== 'advanced'` early-return and threads a new `showShapes` flag
(`canvas.tier === 'advanced'`) down to gate the expand caret, shape
sub-rows, and "Add Shape" toolbar button. No store-layer changes needed —
`addLayer`/`removeLayer`/`reorderLayer`/`duplicateLayer`/`mergeLayerDown`/
`setLayerProps`/`setLayerFrameVisibility` were already tier-agnostic.
Since `selectionKind` can only flip to `'shape'` via a `ShapeRow` click,
and Pixel tier never renders those rows, the shared toolbar naturally stays
locked to its layer-action branch with no extra gating needed there.

Browser-verified live via the `pixelyph` dev server: created a Draw
project, opened the Layers tab in (default) Pixel tier and confirmed no
expand caret / no Add Shape button; painted on a second, newly-added active
layer and confirmed the first layer's own thumbnail content was unaffected
(Checkpoint 1's fix, visible for the first time in the UI once layers were
addressable); merged the two same-color layers down and confirmed no
console errors and a clean single-layer result.

### Checkpoint 4 — Rename tier labels to Pixel/Shape (`c7838b0`)

`ContextBar.jsx` gained `TIER_LABELS`/`TIER_TOOLTIPS` maps (display text

("gradients, stroke, effects, and multiple shapes per layer are lost —
layer count/order/names are preserved") instead of the old "collapses every

Grepped the rest of `src/ui` for other user-visible "Simple"/"Advanced"

Browser-verified: tier buttons read "Pixel"/"Shape" after reload; the

(both advanced-only), with no console errors.
