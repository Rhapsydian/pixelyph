## Work Completed

### Auditing the 40 skips against existing coverage

Before rewriting anything, read every skip alongside `Canvas.js`/`Grid.js`/
`autoLayerSync.js`/`selection.js`/`projectFile.js`'s actual current
implementations. This surfaced that Sessions 1‚Äì3 had already written a
large amount of replacement coverage directly below most of the skips they
left behind (visible in `git log` as tests added alongside the rewrites
that made the old ones obsolete) ‚Äî the skips just hadn't been deleted yet.
Of the 40:

- **31 were already fully superseded** by existing tests and were deleted
  outright, with a short comment pointing at the replacement test's name
  rather than duplicating coverage: 13 in `Canvas.test.js`, 8 in
  `autoLayerSync.test.js`, 4 in `Layer.test.js` (2 more needed a small
  backfill, below), 3 in `selection.test.js`, 4 in `projectFile.test.js`
  (including the full v1/v2 ‚Üí v3 migration tests ‚Äî crop-to-bounds,
  Simple-tier collapse ‚Äî which were already written in detail).
- **9 needed genuinely new tests**, either because behavior itself had
  changed (not just relocated) or because no equivalent existed yet:
  - `addLayer` no longer eager-creates a full-canvas grid ‚Äî new test
    asserts it creates an empty, active layer with `grids: []`.
  - `advanced-tier paintCell` no-ops on `layer.locked` ‚Äî distinct from the
    already-tested `grid.locked` case, needed its own test.
  - `convertTier` simple ‚Üí advanced on a blank canvas ‚Äî the pre-migration
    version auto-filled a solid black layer; the new model has no
    Layer-level style at all (style only exists once a Grid is created by
    painting), so this is a real behavior change, not a relocation. New
    test documents the actual new behavior (empty layer, no grids, nothing
    painted) rather than asserting the retired one.
  - `duplicateLayer` copying a gradient fill independently ‚Äî no equivalent
    existed against the new Grid-styled shape; added, mirroring the
    existing solid-fill duplication test.
  - `addFrame` at an explicit index (not just the default insert-after-
    active) ‚Äî rewrote against `frame.grids` instead of `frame.pixels`.
  - `colorAt`/`topVisibleLayerAt` reading the active frame's own shapes,
    not always frame 0 ‚Äî rewrote against the Grid shape.
  - Two `Grid.test.js` backfills: `growGridToInclude` growing in only one
    negative direction (the old `Layer.test.js` had this; `Grid.test.js`'s
    otherwise-thorough coverage was missing it), and growth on an unfilled
    shape leaving it still empty (the old `isEmpty`-after-`growToInclude`
    equivalent).
  - `resolveActiveGrid` with no prior selection at all (`prevGrid: null`,
    non-empty grid list) ‚Äî the existing four `resolveActiveGrid` tests
    covered id-match, style-match-fallback, mismatch-fallback, and the
    empty/missing-layer null cases, but not this fifth path through the
    function.

Net effect: total test count actually *dropped* (357 ‚Üí 326, since most
skips were deletions rather than 1-for-1 rewrites), while pass count rose
and skip/fail counts both hit zero ‚Äî a cleaner outcome than mechanically
duplicating coverage that Sessions 1‚Äì3 had already written.
