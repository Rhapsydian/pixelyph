## UX Polish Round

### Character Map

**Placeholder thumbnails** (`ee0ba72`): Empty cells in the character map grid now render the Unicode character itself rather than a blank space, making it immediately clear which codepoint each placeholder represents.

**"None" preset option** (`437e7fc`): The wizard's initial charset selector now has "None" as the first/default option, so users can start with an empty map.

**Removed Digits preset** (`2a607f4`): The Digits preset (U+0030–U+0039) was redundant with Basic Latin and was removed.

**Renamed Assign → Create** (`7d63fb0`): The "Assign" button was renamed to "Create" (statically — no Edit variant). It is enabled only when the selected codepoint is new. Hint text added: "Type a character or click a placeholder, then click Create to start drawing it." The "no glyph" message updated to reference the character map rather than a vague "glyph panel."

**Grid cell visual states** (`aa84f35`):
- Glyph cells get a brighter solid border (#666) vs. empty cells (transparent border)
- Clicking a placeholder before clicking Create shows an orange pending-create highlight
- A × remove button appears on hover over glyph cells (red, 14×14, confirms before calling `removeGlyphAction`)
- Clicking any cell (glyph or placeholder) populates the text input with that character

### Glyph Panel

**Consistent delete button** (`73e9adf`): The delete button in `GlyphSetPanel` was made consistent with the character map's × button — hover-only, red (#c0392b), confirms before removing. All glyph cells now have a visible border (#666), with blue highlight for the active glyph.

### Icon Glyph Unicode Field (`1a0bfa9`)

Icon glyphs gained an optional Unicode character association — useful for mapping icon glyphs to existing standard characters for accessibility and cross-app interoperability.

- `unicode` field added to `createGlyph` and serialized in project files
- `parseUnicodeInput(text)` parses: single character, `U+xxxx`, `&#xNNNN;`, `&#NNNN;`, `&name;` (via textarea decode trick for named HTML entities) — returns `null` (clear), a codepoint number (valid), or `undefined` (invalid)
- `updateGlyphMeta(codepoint, patch)` replaces `renameGlyph` — handles `{ name?, unicode? }` patches
- Add form: name input + unicode input with validation, error text, and decoded preview
- Edit section (below glyph browser): live-updates both name and unicode via `updateGlyphMeta`

### Draw Mode

**Stroke linecap removed** (`dd45f4c`): The cap option (butt/round/square) was removed from the stroke editor. Since pixelloom's `gridToPath` only generates closed paths (always Z-terminated), there are never open endpoints where linecap applies.

**Grid overlay offset alignment** (`e26abc9`): In advanced mode, when the active layer has a non-zero offset, the grid overlay now shifts to align with that layer's pixel boundaries via `patternTransform="translate(x,y)"` on the SVG `<pattern>`.

**BrushCursor offset alignment** (`2ba7c3c`): `clientToCell` now snaps the cursor to `Math.floor(px - ox) + ox` rather than `Math.floor(px)`, aligning it with the same offset-relative grid as the overlay.

**Layer offset UI hidden** (`1851d64`): The X/Y offset inputs in `LayersPanel` were removed pending investigation of fractional-offset edge cases. The underlying model field, store action (`setLayerOffset`), grid overlay logic, and cursor snap formula remain intact.
