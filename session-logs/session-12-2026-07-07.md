## Work Completed

**Pixel hover highlight lingers after the pointer leaves the canvas**
(`07dd859`) — `SvgPixelEditor.jsx`'s `cursorCell` state, which positions the
`<BrushCursor>` overlay, was only ever *set* (on pointer down/move) and
never cleared; the canvas `<svg>` had `onPointerDown`/`onPointerMove`/
`onPointerUp` handlers but no leave handler, so the last-hovered cell's
highlight stuck around after the mouse left the canvas. Added a
`handlePointerLeave` that calls `setCursorCell(null)`, wired to the SVG's
new `onPointerLeave` prop. Verified directly in the browser via
`preview_eval`: dispatching a real `pointermove` shows the highlight
`<rect>` (`stroke="#ff5f5f"`), and — since the canvas uses
`setPointerCapture` on drag start — confirmed the *correct* underlying
event to synthesize for React's `onPointerLeave` is `pointerout` with a
`relatedTarget` outside the SVG, not a raw `pointerleave` dispatch (React
derives enter/leave synthetically from `pointerover`/`pointerout`, so a
directly-dispatched `pointerleave` event never reaches React's handler).
Confirmed the highlight clears on that event and reappears correctly on
the next hover.
