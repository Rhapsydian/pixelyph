## Goal

Implement Phase 1 of the plan: a fully working Draw mode at the simple tier — all tools, undo/redo, export (SVG/PNG/WebP), save/load, and autosave — following the plan document from Session 01.

## Key Decisions

**Build order: model → export/IO → UI**
Pure-data layers were implemented first so every piece of logic could be tested with `node --test` before any React code was written. This mirrors pixelloom's own conventions and avoids mixing testable logic with rendering concerns.

**zustand as hot-path coordinator**
During active pointer drags, mutations go through the store's exposed actions, committing a full canvas snapshot to the history stack only on pointer-up. This keeps per-cell writes fast while keeping undo coarse-grained: one stroke = one undo step.

**Live-editing via React ref, not React state**
During pointer drags, the active layer's `<path d="...">` is updated via a ref rather than triggering React's render cycle. Other layers are left untouched. The React render fires only on pointer-up (history commit). This keeps painting smooth at pixel-art scale.

**`autoLayerSync` enforces color-layer mutual exclusivity**
Each painted cell is cleared from any other auto-managed layer before being set in the target color's layer. An auto-layer left entirely empty after a stroke is garbage-collected. This makes the simple tier's "no visible layers" experience work correctly while the model still uses per-color layers underneath.

**Bucket fill: two-pass to fix mirror-mode bug**
The initial single-pass BFS flood-fill was corrupted by symmetry mode — writing a mirror twin mid-BFS would corrupt the "does this cell match the target color?" check on unvisited cells, cutting the fill short. Fixed by splitting into a read-only region-finding pass, then a separate paint pass over the collected region.

**`src/io/platform.js` abstracts environment**
A single interface with two implementations: File System Access API + download-link fallback (web) and IPC (Electron, deferred to Phase 6). Nothing else in the codebase branches on environment.

**Autosave via IndexedDB**
Debounced snapshot on every committed action, restored on next load. A `window.confirm` fires on load if a snapshot exists — explicitly flagged as a rough edge to be replaced by a proper startup screen in Phase 4.

**Testing policy**
All automated tests target only `src/model/`, `src/export/`, and `src/io/` — zero component or SVG rendering tests. UI correctness verified manually per the plan's demo criteria.
