## Work Completed

### Session 1 — Model layer

Rewrote the model against `docs/data-model.md`'s spec: `Grid.js` gained the
styled, auto-cropped Shape type (`createShapeGrid`/`growGridToInclude`/
`shrinkGridToFit`/`mergeGridDown`/`stylesEqual`); `Layer.js` shrank to pure
identity (`{id,name,locked,opacity,frames}`); `Canvas.js`'s `paintCell`,
`colorAt`, `topVisibleLayerAt`, `mergeLayerDown`, `eraseFromLayer`,
`resizeCanvas`, `convertTier`, and the whole frame add/duplicate/remove/
setActive family were rewritten against `layer.frames[i].grids`, plus a new
`activeGridId` pointer and `resolveActiveGrid` (id-match -> style-match ->
first-shape fallback); `autoLayerSync.js` collapsed Simple tier to a single
style-scanned layer; `selection.js`'s `extractRectFromActiveLayer` was
rewritten (the rest of that file needed no change, already built on
`Canvas.js`'s public API); `GlyphSet.js`'s `canvasToGlyphPixels` now expands
a cropped Grid back to full glyph size; `projectFile.js` got a real v3
migration (v1/v2 saves crop each frame to one Grid; Simple-tier saves
collapse their per-color auto-layers into one Layer with multiple Grids).

Two adaptations weren't explicitly named in `docs/data-model.md`'s
Session-1 file list but were necessarily touched since they directly
depend on removed fields: `Canvas.js`'s `convertTier` (read/wrote the
retired `simpleTier`/`autoManaged`) and `resizeCanvas` (assumed a
"full-canvas layer" concept that no longer applies — every Grid now just
shifts its own offset by the anchor delta, which is actually simpler than
before).

### Session 2 — Export/compose

Traced all five files `BACKLOG.md` named (`composeLayersSvg.js`,
`animatedSvg.js`, `spriteSheet.js`, `animatedRaster.js`, `spriteArchive.js`)
before writing anything, and found the scope was much narrower than the
backlog entry implied: only `composeLayersSvg.js`'s `composeLayersBody`
actually reads layer/frame pixel or style data. The other four only ever
call `composeFrameBody`, so fixing that one function cascaded to fix all
four with zero source changes to them — confirmed by reading every call
site, not assumed.

Rewrote `composeLayersBody` to iterate `layers -> frame.grids -> each
visible grid`. Design choice: kept the layer-level `<g id="layer-{slug}">`
wrapper exactly as before (still keyed on `layer.name`, still carries
`layer.opacity`) since it's a documented, already-shipped feature
("exported SVG layers get a CSS-selectable id derived from the layer
name" — `README.md`), and moved `transform`/fill/stroke/effects/opacity
down onto each shape's own `<path>` (SVG allows all of those as direct
presentation attributes on `<path>`, so no extra per-shape `<g>` nesting
was needed). Gradient/filter def ids moved from `grad-${layer.id}` to
`grad-${grid.id}` — simpler than before, since Grid ids are already
globally unique.

An initial Explore agent's proposed design was rejected after cross-
checking it against the actual (pre-Session-1) test expectations: it
suggested keying the exported `<g>` id off the *shape's* name instead of
the layer's, which would have silently broken the layer-name CSS-id
feature and produced an awkward result for Simple tier (whose single
layer is always named "Layer 1", not a color) — caught by tracing where
the old test's expected id (`layer-ff0000`) actually came from (the
pre-migration auto-layer's `name` field, set to the color) rather than
accepting the agent's literal-preservation suggestion at face value.

Updated `composeLayersSvg.test.js` (19 tests unskipped, several requiring
real rewrites, not just removing `.skip`): the "multiple colors" test
changed from expecting 2 `<g>`s to 1 `<g>` with 2 `<path>`s, since Simple
tier is now one Layer with multiple Grids, not multiple auto-layers; added
a new test for two *layers* still getting distinct `<g>`s (the case the
old test name was actually about) and one for a single layer holding
multiple shapes at once. Updated `animatedSvg.test.js` (5 tests): the
def-dedup test needed a real behavioral rewrite, since dedup now happens
by Grid id rather than the old frame-invariant Layer id — switched from
two independently-painted frames to `duplicateFrame` (which preserves a
shape's id across frames), the case that actually still dedupes under the
new model. `spriteSheet.test.js`/`animatedRaster.test.js`/
`spriteArchive.test.js` needed no changes — confirmed they only exercise
pure helpers, never the DOM-dependent SVG-render-then-rasterize path, so
they were unaffected by the whole migration.

Test suite after Session 2: 313 passing / 44 skipped / 0 failing.

## Testing

Both sessions verified via `npm test` only — no browser/preview
verification, since the app is intentionally non-functional end-to-end
until Session 3 wires the UI to the new model (this was true before this
session and remains true after it; not a regression). Iterated file-by-
file: after each source rewrite, ran the affected test file in isolation
(`node --test test/path/to/file.test.js`) to triage exactly which
assertions needed skipping vs. rewriting vs. leaving alone, then ran the
full suite at the end of each redesign-phase to confirm a clean
pass+skip count before committing.
